Address Taken

Time limit 2 seconds
Memory limit 64.0 MB
Input stdin or input.txt
Output stdout or output.txt

The country of Berland finally has internet access! There are two email services available: Bmail and Bandex (initially nobody is registered on either of them). All N N N citizens of Berland now want email addresses. Each of them has already come up with their preferred email address, but, unfortunately, some of these addresses might have coincided. All Berlanders attempt to register their addresses in turn, and each of them follows this algorithm:

  • first the Berlander tries to register their address on Bmail (since this service is considered more prestigious),
  • if the address has already been registered there, they try to register it on Bandex,
  • if it has been registered there as well, they give up.

For each Berlander determine if they will be able to register their address, and on which service.

Input format

An integer N N N ( 1 ≤ N ≤ 1 0 5 1 \leq N \leq 10^5 1≤N≤105) is given in the first line, followed by N N N nonempty strings — the desired email addresses of the Berlanders in the order they will register them in (each on a separate line). The addresses may contain lowercase Latin letters and digits. The total length of the addresses does not exceed 1 0 5 10^5 105 characters.

Output format

For each of the N N N Berlanders output (on a separate line) BMAIL or BANDEX if they will register on the corresponding service, or FAIL if they won't be able to register on either.
